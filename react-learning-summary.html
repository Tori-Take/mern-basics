2. 今回の学習内容と結果

Reactの最も基本的な3つの概念を中心に、サンプルプログラムの作成と解説を通じて学習しました。

  • コンポーネント (Component): UIを構成する再利用可能な「部品」であるという概念を理解しました。Greeting.jsx, LikeButton.jsx, Counter.jsxを作成し、部品化のメリットを体験しました。
  • Props (プロップス): 親コンポーネントから子コンポーネントへデータを渡す仕組みを学びました。これにより、コンポーネントの再利用性が高まることを確認しました。
  • State (ステート): コンポーネントが内部で持つ「状態」であり、useStateフックを使って管理することを学びました。Stateが更新されるとUIが自動で再描画される、Reactのインタラクティブな動作の核となる部分を実践しました。
  • JSX: JavaScript内でHTMLのようにUIの構造を記述する構文に慣れました。classNameやonClickなどの基本的な使い方を習得しました。
  • イベントハンドリング: onClick属性に関数を渡すことで、ユーザーの操作に応じた処理を実行する方法を学びました。
  • 状態のリフトアップ (Lifting State Up):

    複数の子コンポーネントで状態を共有するために、状態を共通の親コンポーネントに引き上げる（リフトアップする）方法を学びました。これにより、コンポーネント間の連携が可能になることを確認しました。

  • コールバック関数による親子間の連携:

    親が子に「関数」を渡し、子のイベント（クリックなど）をきっかけに親の関数を実行させることで、子が親の状態を変更する仕組みを実装しました。

  • リストの動的レンダリング:

    JavaScriptの.map()メソッドを使い、データの配列からコンポーネントのリストを動的に生成する方法を学びました。これにより、より柔軟でデータ駆動なUI構築が可能になりました。

  • Reactにおける状態更新の不変性 (Immutability):

    State（特に配列やオブジェクト）を更新する際は、直接変更するのではなく、スプレッド構文([...array])などを使ってコピーを作成してから変更する、というReactの重要な原則を学びました。

結果

Reactの基本的な考え方に加え、複数のコンポーネントが連携する際のデータフローや状態管理の設計思想について深く理解されました。ご自身でコードの意図を読み解き、応用的な質問もできるレベルに到達しています。

3. 次回の学習項目

今回の基礎を元に、より実践的なアプリケーションを構築するための次のステップです。

  • 副作用を扱う (useEffectフック):

    コンポーネントのレンダリング以外の処理（APIからのデータ取得、DOMの直接操作、タイマー処理など）を扱うためのuseEffectフックについて学びます。

  • フォームの扱い方:

    入力フォーム（<input>, <textarea>など）の値をStateで管理し、ユーザーの入力をハンドリングする方法を学びます。
