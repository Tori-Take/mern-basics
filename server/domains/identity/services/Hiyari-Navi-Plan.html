開発計画書

ヒヤリハット分析システム「Hiyari-Navi」

1. プロジェクト概要

組織内で発生する「ヒヤリハット」を簡単に記録・集約し、それらのデータを可視化・分析することで、潜在的なリスクを特定する。これにより、組織全体の安全意識を向上させ、重大な事故を未然に防ぐことを目指す。

2. 技術スタック

TODOアプリ開発で培った経験を最大限に活かしつつ、「分析」というテーマに合わせて新しい技術を導入します。

  • バックエンド: Node.js, Express.js
  • フロントエンド: React
  • データベース: MongoDB
  • UIフレームワーク: React-Bootstrap
  • データ可視化: Chart.js (react-chartjs-2)

3. 主要機能 (MVP)

アプリケーションの核となる、実用最小限の製品（MVP）として以下の機能の実装を目指します。

機能分類 機能名 詳細
認証 ユーザー登録・ログイン TODOアプリの認証基盤を流用・発展させ、ユーザーを管理する。
投稿 ヒヤリハット投稿 日時、場所、詳細、カテゴリ、タグを記録できるフォーム。
閲覧 投稿一覧 自分が投稿したヒヤリハットを一覧で確認・検索できる。
分析 ダッシュボード 全ての投稿データを集計し、グラフで可視化する。（管理者向け）

4. データベース設計 (Mongoose スキーマ)

アプリケーションの心臓部となるデータ構造です。

// HiyariHatto.model.js
const hiyariHattoSchema = new mongoose.Schema({
  user: { type: mongoose.Schema.Types.ObjectId, ref: 'User', required: true },
  tenant: { type: mongoose.Schema.Types.ObjectId, ref: 'Tenant', required: true },
  incidentDate: { type: Date, required: true },
  location: { type: String, trim: true },
  details: { type: String, required: true },
  category: { type: String, required: true, enum: ['転倒・つまずき', '衝突', '誤操作', '火災・感電', 'その他'] },
  tags: [String],
}, { timestamps: true });

5. 開発ロードマップ

フェーズ1：バックエンド基盤の構築 (API開発)

目標: ヒヤリハットを保存し、読み出すための「土台」を完成させる。

  1. プロジェクトセットアップ
  2. モデル作成 (`HiyariHatto.model.js`)
  3. APIエンドポイント作成 (`POST /api/hiyarihattos`, `GET /api/hiyarihattos`)
  4. ルーティング設定 (`hiyarihattos.routes.js`)
フェーズ2：フロントエンド実装 (投稿と一覧表示)

目標: ユーザーが実際にヒヤリハットを投稿し、閲覧できる画面を完成させる。

  1. 投稿フォーム作成 (`HiyariHattoCreatePage.jsx`)
  2. 投稿機能の実装 (API連携)
  3. 一覧表示ページ作成 (`HiyariHattoListPage.jsx`)
フェーズ3：分析機能の実装 (ダッシュボード)

目標: 蓄積されたデータを「見える化」し、分析の第一歩を踏み出す。

  1. データ可視化ライブラリ導入 (`Chart.js`)
  2. 集計APIの作成 (`GET /api/hiyarihattos/stats`)
  3. ダッシュボード画面作成 (`DashboardPage.jsx`)
フェーズ4：認証と権限管理

目標: TODOアプリで実装した高度な認証・権限機能を統合し、アプリを完成させる。

  1. 認証機能の統合 (`auth`ミドルウェア適用)
  2. 権限管理ロジックの実装 (編集・削除権限など)
