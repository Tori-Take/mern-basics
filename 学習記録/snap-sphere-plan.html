写真投稿アプリ『Snap-Sphere』

1. プロジェクトの目的とコンセプト

  • アプリ名: Snap-Sphere (スナップスフィア)
    • 「Sphere」には「範囲・領域」という意味があり、共有範囲を柔軟に設定できるこのアプリのコンセプトに合致します。
  • 目的: 現場の写真、イベントの記録、共有したい風景などを、タイトル・説明・位置情報といった「コンテキスト」と共に、組織内の適切な範囲に簡単かつ安全に共有する。
  • ゴール:
    1. ユーザーがスマートフォンやPCから写真を安全にアップロードできる。
    2. 投稿ごとに、柔軟な公開範囲（非公開、部署内、組織内、特定ユーザーなど）を設定できる。
    3. 投稿された写真が、時系列や地図上で閲覧できる、直感的で楽しいUIを提供する。

2. データベース設計 (Mongoose スキーマ)

アプリケーションの心臓部となる、投稿データPostの設計です。

// server/domains/snapsphere/post.model.js (新設)
const postSchema = new mongoose.Schema({
  // --- 基本情報 ---
  title: { type: String, required: true, trim: true },
  description: { type: String, trim: true },
  photo: {
    public_id: { type: String, required: true }, // CloudinaryでのファイルID
    secure_url: { type: String, required: true }, // HTTPSでの画像URL
  },
  
  // --- コンテキスト情報 ---
  shotDate: { type: Date, default: Date.now }, // 撮影日時
  location: {
    type: {
      type: String,
      enum: ['Point'], // GeoJSON形式
      default: 'Point',
    },
    coordinates: {
      type: [Number], // [経度, 緯度] の順
      index: '2dsphere' // 地図検索のためのインデックス
    },
    address: { type: String, trim: true } // (任意) 住所文字列
  },

  // --- 投稿者と所属 ---
  postedBy: { type: Schema.Types.ObjectId, ref: 'User', required: true },
  tenantId: { type: Schema.Types.ObjectId, ref: 'Tenant', required: true },

  // --- ★★★ 公開範囲設定 ★★★ ---
  visibility: {
    type: String,
    required: true,
    enum: ['private', 'department', 'tenant', 'specific_users', 'public'],
    default: 'department',
  },
  // visibilityが 'specific_users' の場合に、許可されたユーザーIDのリスト
  allowedUsers: [{ type: Schema.Types.ObjectId, ref: 'User' }],

}, { timestamps: true });

3. 技術的アプローチとアーキテクチャ

  • 画像ストレージ (Cloudinary):
    • フロントエンドから直接Cloudinaryに画像をアップロードする「署名付きアップロード」方式を採用します。
    • この方式により、画像データが私たちのサーバーを経由しないため、サーバー負荷を大幅に軽減できます。
  • 位置情報:
    • 写真のEXIF情報から自動で取得するか、ユーザーが地図上でピンを指して設定できるようにします。（初期実装では手動入力を想定）
  • バックエンド (API):
    • server/domains/snapsphere/ ディレクトリを新設し、post.model.js, post.routes.js, post.controllers.js を作成します。
    • ヒヤリハットアプリと同様に、CAN_USE_SNAPSPHEREという新しい利用権限を定義します。
  • フロントエンド (UI):
    • client/src/features/snapsphere/ ディレクトリを新設し、pages, components を作成します。
    • ヒヤリハットアプリのUI構成を参考に、投稿フォームと投稿一覧（ギャラリー）ページを構築します。

4. 段階的実装ロードマップ (Phased Roadmap)

この新しいアプリを、以下の4つのフェーズに分けて、一歩ずつ着実に構築していきましょう。

フェーズ1：バックエンド基盤の構築

目標: 画像をアップロードし、投稿を保存・取得するための「土台」を完成させる。

  1. Cloudinaryのアカウントを作成し、APIキーを.envファイルに設定。
  2. Postモデル (post.model.js) を作成。
  3. バックエンドに、署名付きアップロードのためのAPI (POST /api/snapsphere/upload-signature) を作成。
  4. 写真投稿の基本API (POST /api/snapsphere/posts) と一覧取得API (GET /api/snapsphere/posts) を作成。（この段階では、まだ複雑な公開範囲のロジックは含めない）

フェーズ2：フロントエンドのMVP実装

目標: ユーザーが実際に写真を投稿し、閲覧できる最小限の画面を完成させる。

  1. 写真、題名、説明などを入力できる基本的な投稿フォーム (CreatePostPage.jsx) を作成。
  2. ★カメラ機能の実装: 投稿フォームに「ライブラリから選択」と「カメラで撮影」の2つのボタンを設置し、どちらからでも画像をアップロードできるようにする。
  3. フォームが送信されたら、フェーズ1で作成したAPIを呼び出して、CloudinaryへのアップロードとDBへの保存を実行するロジックを実装。
  4. 投稿された写真が一覧で表示される、シンプルなギャラリーページ (PostListPage.jsx) を作成。

フェーズ3：高度な機能の実装

目標: アプリの核となる「共有」と「位置情報」の機能を実装する。

  1. 公開範囲設定:
    • 投稿フォームに、公開範囲（部署内、組織内など）を選択するUIを追加。
    • バックエンドの一覧取得APIに、ユーザーの権限に応じて閲覧可能な投稿だけを返すロジックを実装。
  2. 位置情報:
    • 投稿フォームに、地図上で場所を選択できるコンポーネント（react-leafletなど）を導入。
    • 投稿一覧に、地図上で写真がマッピングされるビューを追加。

フェーズ4：統合と洗練

目標: アプリケーションを既存のシステムに完全に統合し、完成度を高める。

  1. App.jsxに新しいルートを追加し、PermissionRouteで保護。
  2. Navbar.jsxやダッシュボードに、「Snap-Sphere」へのリンクを追加。
  3. 全体のUI/UXを磨き込み、レスポンシブデザインを調整。
